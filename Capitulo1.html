Exercicios de HTML

1.1-Quais são os 3 tipos de códigos básicos de uma página de hipertexto? Explique para que serve cada um dos tipos

R: 3 tipos de codigo basico de hipertexto são: HTML,CSS e Javascript. O HTML responsavel pela estruturação de elementos dentro da arquivo, o css estiliza a pagina, dando cores, e elementos mais esteticamente agradavel a pagina , e o Javascript que é o terceiro, Javascript já se trata de uma linguagem de programação, diferente dos outros, pois já consegue manipular dados fornecidos para a página.

1.2-Quantos arquivos CSS e Javascript podem estar vinculados a uma mesma página de hipertexto? Justifique sua resposta.

R:Não há um limite especifico para associar o numero de arquivos css e Javascript que podem ser associados na mesma pagina de hipertexto.Uma das praticas mais comuns é organizar o código em várias arquivos para facilitar o acesso e a manutenção.

1.3-É possível diferentes páginas de hipertexto terem vínculo com os mesmos arquivos Javascript e CSS? Justifique sua resposta

R:Sim,como dito na respota anterior, não há um limite para associação de arquivos em paginas de hipertexto. Além de ser reutilizavél, ao vincular os mesmos arquivos css e Javascript, alterções feitas nesses arquivos, refletiram nas em todas as paginas de HTML vinculadas, simplificando assim o tempo de manutenção.

1.4-Descreva a diferença entre uma página de hipertexto estática e uma dinâmica, pontuando as vantagens e desvantagens de cada um dos dois tipos.

R:Basicamente, as paginas estaticas tem conteudo fixo e são faceis de criar, por conta disso, são mais limitadas em questão de conteudo. podendo ter pouca interatividade e personalização. Paginas dinâmicas são criadas sobre demanda,permitindo uma interatividade e personalização maior, porém são mais complexas e requerem uma hospedagem especifica.

1.5-Qual é o caminho percorrido por uma requisição HTTP a uma página de hipertexto estática?

R:O processo de uma requisição HTTP para uma página de hipertexto estática é o seguinte: O navegador inicia a requisição, resolve o DNS para obter o endereço IP do servidor, estabelece a conexão, envia a solicitação GET para o servidor. O servidor processa a solicitação, enviando o conteúdo da página de volta. O navegador então renderiza a página e a exibe para o usuário. O navegador também pode optar por armazenar em cache a página para acessos futuros.

1.6-Qual é o caminho percorrido por uma requisição HTTP a uma página de hipertexto dinâmica?

R:Uma requisição HTTP para uma página de hipertexto dinâmica envolve o navegador iniciando a solicitação, resolvendo o DNS para obter o endereço IP do servidor, estabelecendo a conexão, e enviando a solicitação GET. O servidor processa a solicitação, gera o conteúdo da página com base em dados dinâmicos, e envia o conteúdo de volta ao navegador. O navegador então renderiza a página e a exibe ao usuário.

1.7-Quais as vantagens da atualização parcial de conteúdo em segundo plano em uma página de hipertexto usando Javascript?

R:Atualizar partes específicas de uma página de hipertexto em segundo plano com JavaScript traz vantagens como uma experiência do usuário mais fluida, economia de banda, redução da carga do servidor, personalização dinâmica, interatividade melhorada e suporte a aplicativos web em tempo real.
